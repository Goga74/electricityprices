1. Сборка Docker-образа
Перейдите в каталог, где находится ваш Dockerfile и выполните следующую команду:Codedocker build -t your-app-name .Здесь your-app-name — это имя, которое вы присвоите своему Docker-образу.2. Запуск Docker-контейнера локально
После успешной сборки образа вы можете запустить контейнер:Codedocker run -p 8074:8074 your-app-name
-p 8074:8074 перенаправляет порт 8074 внутри контейнера на порт 8074 на вашей локальной машине. Убедитесь, что этот порт не занят другими приложениями.
your-app-name — имя вашего Docker-образа.

3. Проверка работы приложения

Доступ к приложению:
Откройте браузер и перейдите по адресу http://localhost:8074.
Проверьте, что приложение работает корректно.




Просмотр логов контейнера:
Вы можете увидеть логи контейнера, используя команду:Codedocker logs <container-id>
Чтобы узнать container-id, используйте:Codedocker ps



4. Завершение работы контейнера
Если вы хотите остановить контейнер, используйте:Codedocker stop <container-id>Где <container-id> — это идентификатор вашего контейнера.Дополнительные проверки

Проверка соединения с БД: Если ваше приложение требует подключения к базе данных, убедитесь, что оно настроено правильно и доступно.
Тестирование API: Используйте инструменты, такие как Postman или curl, для тестирования API вашего приложения.
